Own VPN

At my new school, I needed a VPN because the school Wi-Fi blocked many sites. I built my first VPN using WireGuard, but it relied on the UDP protocol, which was blocked on our network. To resolve this, I installed PiVPN and configured OpenVPN to use TCP, which wasn’t blocked. Now, I have a functional VPN on my phone that bypasses the restrictions.

OpenVPN on Chromebook

Adding a VPN on a Chromebook seemed straightforward—just download the OpenVPN app and import the configuration. However, my school account restricted app installations. I used the built-in VPN feature, imported the certificate, and faced issues. After some trial and error, I found a workaround that I won’t disclose here to prevent the school from blocking it. Check back on 01/02/2025 for updates.

Own NAS

A NAS (Network Attached Storage) allows file storage, virtual machine hosting, or running Docker containers. I used it for backing up files from my PC and phone. I started with TrueNAS Core, which is lightweight and user-friendly, and later switched to TrueNAS Scale, which runs on Debian instead of FreeBSD. I repurposed an old laptop for this project, making it an excellent solution for beginners.

Own Dashboard

I created a dashboard to manage and monitor my network and services efficiently. This dashboard streamlined how I interact with my projects and systems, consolidating everything in one place for easier access and control.

Reverse Proxy

Unlike a VPN, a reverse proxy doesn’t assign an IP address from your network. It allows users to access apps securely without being directly on your network. I used Cloudflare for this, with the connector installed on a Raspberry Pi. This setup ensures that my network remains secure while others access specific apps.
